# Consult - Advisorへの相談

最高知能モデル（Fable xhigh）の `advisor` エージェントに相談し、助言を受け取る。

## 手順

1. **相談パッケージを作る**。Advisorはこの会話を見られないため、以下を自己完結でまとめる:
   - **目的**: 最終的に達成したいこと（手段ではなくゴール。XY問題を避ける）
   - **現状**: 何が起きているか。エラーメッセージは全文を貼る
   - **試行履歴**: 何を試して何がダメだったか（時系列）
   - **関連情報**: 関係するファイルパス・ブランチ名（Advisorは読み取りツールを
     持つので、パスを渡せば自分で読んで事実確認する）
   - **制約**: 技術・時間・ポリシー上の制約
   - **問い**: 判断してほしいことを具体的な質問形式で。
     「どうすればいい？」ではなく、自分の仮説や選択肢を添えて問う

2. `advisor` エージェントを Agent ツールで起動し、パッケージを渡す

3. **助言を受けたら**:
   - ユーザー向けに要点を報告（診断・推奨・次の一手）
   - 実装が必要なら、助言をそのまま添えてExecutorに再ディスパッチ（`/advisor-mode` 参照）
   - アーキテクチャ等の重要な決定は、作業リポジトリの適切な場所（docs/、plans/ 等）に記録

## 使いどころの規律

Advisorは最高コストのモデル。以下のときだけ呼ぶ:

- 同じエラー・問題で **2回失敗** した
- 複数のアプローチがあり、選択を誤ると手戻りが大きい
- アーキテクチャ・データ設計など影響範囲の大きい判断
- セキュリティ・データ損失などリスクの高い操作の事前確認

以下はAdvisorを使わない:

- ドキュメントを調べれば分かる情報不足 → `researcher` エージェント / context7 / WebSearch
- 仕様の曖昧さでユーザーの意向が必要なもの → ユーザーに直接確認

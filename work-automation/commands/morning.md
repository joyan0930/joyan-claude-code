# 朝のブリーフィング生成

## 処理フロー

### Step 1: プロジェクト一覧取得
`projects/` ディレクトリから active なプロジェクトを列挙

### Step 2: 今日のカレンダー取得
Google Calendar MCP で今日のイベント一覧を取得

### Step 3: プロジェクト別処理 (並列サブエージェント)
各プロジェクトについて:

```
Task: {project.name} の状態更新

1. sources.yaml を読み込み
2. 今日のミーティングでこのプロジェクトに関連するものを特定
   (calendar_pattern でマッチング)
3. 関連 Docs を取得し、前回からの更新を確認
4. Slack チャンネルから直近24時間の重要メッセージ抽出
5. 以下を更新:
   - state/decisions.md: 新しい意思決定
   - state/actions.md: アクションアイテム (追加・完了・期限変更)
6. サマリーを返す
```

### Step 4: 統合ブリーフィング生成
